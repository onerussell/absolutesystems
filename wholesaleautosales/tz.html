Project Name : Wholesale Auto Sales

Администрантивный интерфейс состоит из следующих основных модулей.
  • Система управления инвенторией (Inventory)
  • Система управления зарегистрированными пользователями (Users)
  • Система оповещения пользователей о новых поступлениях (Notifications)
  • Система листования позиций инвентории на Ebay

Система управления инвенторией

Доступна администратору сайта (1)
Администратор сайта :
  • имеет доступ (добавление, редактирование, удаление, сортировка, поиск по определенным полям)(2) к полному списку автомобилей находящихся в инвентории.(3)
  • имеет возможность выбирать из всей инвентории 2 машины, которые будут отображаться в разделе featured cars.
  • имеет возможность (добавлять, удалять) избранные машины в раздел "specials" которые затем отображаются в одноименном разделе на сайте.
  • имеет возможность добавлять неограниченное количество фотографий автомобиля и изменять порядок их следования. Фотографии на сайте отображаются с помощью специального флэш компонента http://www.engine37.com/e37zoom/index1.html

Система управления зарегистрированными пользователями

Доступна администратору сайта
Администратор сайта :
  • имеет доступ к полному списку зарегистрированных пользователей сайта. Здесь опять же актуально все сказанное в примечании (2)
  • имеет возможность навигации (google type pager), поиска и сортировки по таблице зарегистрированных пользователей (2)
  • имеет права на редактирование, удаление пользователя (добавление нового пользователя осуществляется в процессе регистрации на вебсайте самим пользователем)
  • имеет возможность отправлять стандартное письмо выбранному пользователю с линком на ту или иную машину в инвентории. (например http://www.engine37.com/wholesaleautosales/4/car.php?car_id=124)
  • имеет возможность отправить зазипованый архив с фотографиями той или иной машины. (эта функциональность во многом повторяет функциональность из "Системы оповещения пользователя о новых поступлениях" )
  • имеет доступ (сортировка, поиск, удаление) к таблице заявок оставленных зарегистрированными пользователями таблица заявок включает в себя следующие поля (имя пользователя, email пользователя, дата заявки, производитель (Ford), марка (Explorer), ограничения на год выпуска (с 1996 по 2002)

Система оповещения пользователей о новых поступлениях

Зарегистрированный пользователь может оставить заявку на получение специальных оповещений
Предполагаются 3 различных вида оповещений :
  • e-mail с линком на фотографии машины
  • email c присоединенным архивом фотографий машины
  • сообщение на сотовый телефон. Технически сводиться к отправке еще одного email на адрес вида "номер телефона"@"имя сотового оператора".
Последовательность событий такая. Скажем человека интересует марка "Ford" модель "Explorer" модельный ряд старше 1997 года). Сделать подобного рода заявку пользователь может на странице http://www.engine37.com/wholesaleautosales/4/inventory.php кликнув на линк "notify" и заполнив соответствующую форму. Соответственно после авторизации у зарегистрированных пользовательей помимо прочих будет еще секция "my requests" . Там будет выводиться таблица заявок из которой можно будет удалять записи и в которую можно будет добавлять записи при помощи все той-же формы которая доступна по линку "notify" (4)

Система листования позиций инвентории на Ebay

Администратор сайта должен иметь возможность
  • лиcтовать товар используя вызовы к Ebay API
  • изменять глобальные настройки связанные с ebay аккаунтом (ebay user name, ebay passwd, ebay licence key etc)
  • изменять html шаблон аукциона использующийся при листовании(5)

Примечания

(1) В данном проекте на админской стороне нет иерархии пользователей как таковой. Есть всего один юзер "Admin" который является полновластным хозяином сайта. В будующем могут попадаться проекты в которых помимо Admina к административной части имеет доступ также пользователи с урезаными привилегиями для который в некторых модулях нет права на редактирование/удаление в некоторых даже на просмотр.

(2) Эта задачка чаще всего встречается при построении админских интерфейсов. И в даном проекте она встречается минимум 3 раза (вывод таблицы находящихся в инвентории автомобилей, вывод таблицы зарегистрированых пользователей, вывод таблицы заявок зарегистрированых пользователей). Если количество строк в таблице превышает определенное число то вверху таблицы появляется элемент для навигации по набору данных (google like pager). (http://pear.php.net/package/DB_Pager), (http://pear.php.net/package/Pager).
Под заголовком каждого поля необходимо иметь элементы управления отвечающие за сортировку (обычно стрелка вверх - стрелка вниз) и элементы управления отвечающие за фильтрацию набора данных (input)
Cортировка и фильтрация являются необязательными элеменами. Могут включаться и отключаться для каждого поля индивидуально.
То есть я вижу это как черный ящик который на входе получает sql запрос и такие параметры как набор столбцов и их ширина, флаги для сортировки и быстрого поиска - на выходе массив данных который необходим для отрисовки html таблички cо встроеными элементами управления (пейджер, сортировка, фильтрация).

(3) При добавлении автомобиля в инвенторию последовательность действий следующая :
1. Select Make
2. Select Model
3. Select Year
как сделано например на (http://www.jcwhitney.com/). Понятно что здесь имеем дело со связанными списками, То есть при выборе скажем марки Ford у нас остается ограниченое количество моделей разработаных только фирмой форд и сооответственно при выборе определенной модели перед нами список годов когда та или иная модель выпускалась.

(4) Технически задачу можно решить двумя способами. При каждом новом insert'e в таблицу автомобилей, тут же опрашивать таблицу заявок на предмет совпадения критериев и сразу отправлять email. Либо добавить cronjob который каждый час вызывает соответствующий скрипт и из которого происходит рассылка писем.

(5) Речь конечно же идет не о встроеном html редакторе. Шаблоны изготовляются дизайнером по поределенным правилам. Далее набор шаблонов складывается в определенную дирректорию после чего становиться доступным администратору.
